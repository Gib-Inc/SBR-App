Extend this inventory and PO AI system so it incorporates marketing and sales performance data from Google Ads, Meta (Facebook/Instagram) Ads, TikTok Ads, Shopify, and Amazon Seller Central, entirely inside this Replit project. Do not rely on n8n or any external orchestration. All ingestion, aggregation, and UI must live in this app’s backend and frontend.

The goal is: for each finished product, the AI and humans can see a unified view of inventory, sales velocity, and ad performance, and we can later plug this data into the AI forecasting prompts.

In the database/schema, add:

A Channel table that defines canonical channels for this app. At minimum create channels for:

google_ads

meta_ads

tiktok_ads

shopify

amazon

A ProductChannelMapping table that links our internal finished product IDs to external identifiers per channel. Each mapping row must store:

channelId (reference to Channel)

Make these mappings reusable by all ingestion jobs so we do not hardcode external IDs anywhere else.

For SalesSnapshot:

Implement functions to upsert daily sales metrics per product and channel.

Use the existing infrastructure that already sees orders from Shopify and Amazon through Extensiv and/or direct integrations.

Normalize those orders into this SalesSnapshot table so that, for each product and each day, we know unitsSold and revenue by channel (shopify and amazon).

Create a backend scheduler/job mechanism inside this app (follow the existing pattern for any batch jobs) that will periodically:

Refresh AdPerformanceSnapshot for recent days using the stubbed fetchers.

Refresh SalesSnapshot for recent days using order data already available in this system.

Build a ProductForecastContext aggregation

Add a ProductForecastContext view or materialized table that aggregates, per finished product, all the signals needed for forecasting. For each product, compute and store at minimum:

onHandPivot and onHandHildale (from existing inventory/transactions)

inboundUnits (sum of open POs not yet received)

unitsSold_7d and unitsSold_30d (using SalesSnapshot, summed across channels and also broken out by channel)

adSpend_7d and adSpend_30d per ad channel (from AdPerformanceSnapshot)

conversions_7d per ad channel

ROAS_7d per ad channel (where revenue is available)

daysOfStockLeft based on recent sales velocity and current on-hand + inbound

Build this using the existing transaction, PO, and inventory tables as the source of truth. Make ProductForecastContext easily queryable by both the AI layer and the UI.
