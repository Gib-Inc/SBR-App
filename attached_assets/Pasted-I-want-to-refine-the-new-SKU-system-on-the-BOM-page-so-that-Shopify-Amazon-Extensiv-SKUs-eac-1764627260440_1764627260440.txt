C) ENSURE CHANNEL FIELDS ARE CORRECTLY WIRED
========================================
I want a clear health-check that:

- Shopify integration uses product.shopifySku.
- Amazon integration uses product.amazonSku.
- Extensiv integration uses product.extensivSku.
- Mappings in the DB actually correspond to real SKUs in each external system.

1) Code-level wiring check:

   - Confirm (and enforce via code) that:

     - Shopify order + product import logic ALWAYS resolves products via:
       `findProductByShopifySku(line.sku)` and reads/writes `product.shopifySku`.

     - Amazon order import logic ALWAYS resolves via:
       `findProductByAmazonSku(sellerSku)` (or equivalent) and reads/writes `product.amazonSku`.

     - Extensiv inventory sync ALWAYS resolves via:
       `findProductByExtensivSku(itemCode)` and reads/writes `product.extensivSku`.

   - Add small unit/integration tests for each helper to ensure they query the correct fields:
     - e.g. mock product with only shopifySku set and verify Shopify lookup succeeds while others don’t.

2) UI “Verify Channel SKUs” action:

   - Extend the existing “SKU Mapping” UX with a “Verify Channel SKUs” button (either on the BOM toolbar near “SKU Mapping” or inside the mapping dialog).

   - When clicked, run a backend verification routine that for each enabled integration:

     Shopify:
       - For a sample of mapped products (e.g. first 50 where product.shopifySku is NOT NULL):
         - Call Shopify API to try to fetch the variant by its SKU (or appropriate query).
         - Classify each as:
           - OK (external SKU exists and matches)
           - Missing remotely (no variant with that SKU)
         - Also count external SKUs that exist in Shopify but aren’t mapped (if that’s cheaply discoverable).

     Amazon:
       - For a sample of mapped products with product.amazonSku:
         - Call Amazon API (seller SKU endpoint) and verify existence.
         - Classify the same way: OK vs Missing.

     Extensiv:
       - For each product with product.extensivSku:
         - Check Extensiv’s API for an item with that code.
         - Classify OK vs Missing.

   - Present a summary report in the UI, e.g.:

     - Shopify: 45 OK, 3 Missing mappings, 12 unmapped external SKUs.
     - Amazon: 20 OK, 1 Missing, 5 unmapped.
     - Extensiv: 10 OK, 0 Missing, 0 unmapped.

     - House SKU, Product Name, Channel SKU, Status.
     - Include a link or button to jump directly into the mapping wizard for that channel.

3) Integration guardrails (light touch):

   - When a user clicks “Test Connection” or “Sync Now” for Shopify/Amazon/Extensiv:
     - Optionally run a quick version of the verification (small sample) and show warnings if there are many missing or unmapped SKUs.
   - Do NOT block sync if everything else is working; just clearly surface that channel SKUs might need cleanup.
