1) Run end-to-end “golden path” flows with test data
Create a small set of test SKUs and run at least these flows end-to-end INSIDE the running app:

Flow A: New product from scratch
- Create a finished product with BOM (components from stock inventory).
- Assign proper barcodes (GS1 for finished products, internal for raw materials) and verify they print/preview correctly.
- Create a PO for raw materials → Receive it into Pivot/Hildale → verify:
  - Inventory transaction records created.
  - Stock inventory and AvailableForSale update correctly.
  - AI Logs show the movement.
- Produce units (if applicable) and verify forecasting fields update (Hildale/Pivot/AvailableForSale).

Flow B: Sale + AI recommendation
- Create a sales order for that product (or pull one from Shopify/Amazon if the integration is already wired).
- Verify:
  - Inventory decrements correctly (exactly once, no double counting).
  - AvailableForSale updates correctly.
  - AI Logs show the sale movement.
  - After running the decision engine, SKU Recommendations shows a NEW recommendation if stock is low (risk, days until stockout, recommended qty, % off target, qty on PO).

Flow C: Return + inventory correction
- Create a return for that sale using the Returns module.
- Move it through the states (Created → In Transit → Received).
- Verify:
  - The correct transaction is written when the return is RECEIVED.
  - Stock increases at the correct location.
  - AvailableForSale updates when appropriate.
  - AI Logs show the return movement.
  - SKU Recommendations update (risk and recommended qty change after the return).

Flow D: Recommendation → PO
- From SKU Recommendations, take a NEW recommendation and start a PO from it.
- Verify:
  - PO is created inside Suppliers → PO History & Issues (not a new page).
  - Supplier, SKU, and recommended qty are pre-filled.
  - Saving the PO sets the recommendation status to ACCEPTED and links it to that PO.
  - Qty on PO and status columns in SKU Recommendations update correctly.
  - AI Logs record that a recommendation turned into a PO.

2) UI + table consistency check
For each of these pages:
- Barcodes
- Products/BOM + Stock Inventory
- Suppliers → PO History & Issues
- Sales Orders
- Returns
- AI Agent → Data Sources / Rules / SKU Recommendations / Logs

Make the UI consistent with the rest of the app:
- Each section has a clear heading and controls at the top, and the data table in a card/container below it.
- Data tables:
  - Are contained in their card with vertical/horizontal scroll INSIDE the card.
  - The overall page does NOT become horizontally scrollable.
  - Action icons/columns on the right remain visible or behave consistently while scrolling.
- Modals / side drawers:
  - No fields run off-screen horizontally.
  - Buttons and fields show the same structure regardless of how you open them.
- Remove any leftover “debug” text, temp buttons, or inconsistent labels from earlier experiments.

For Extensiv, Shopify, Amazon, QuickBooks, GHL, PhantomBuster, and any Meta/Google Ads work that already exists:
- Make sure:

- Verify Integration Health / key rotation alerts (if implemented) do NOT leak tokens and show sane messages.

4) Fix any issues you find
- For every error, inconsistency, or obvious bug encountered in the flows above:
  - Fix it in the code.
  - Add/adjust tests or basic checks where appropriate (follow existing testing patterns).
  - Keep fixes scoped to stability and correctness; do not add new features.

Do all of this WITHOUT adding new major features. This step is about hardening the existing V1 app so it is stable, consistent, and trustworthy for real-world use.
