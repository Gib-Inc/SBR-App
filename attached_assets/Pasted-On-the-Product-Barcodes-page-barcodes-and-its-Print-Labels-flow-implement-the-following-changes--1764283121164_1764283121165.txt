9) PRINTER STATUS BADGE – REALISTIC BEHAVIOR
==================================================
- Replace “Printer: Ready” behavior with a more honest capability check.

State model:
- Badge can be in one of three states:
  - Neutral: “Printer: Not verified”.
  - Available (green): “Printer: Available”.
  - Error (red): “Printer: Error”.

Logic:
- On modal open:
  - If `window.print` exists:
    - Set badge to “Printer: Available”.
  - If `window.print` is missing or throws when probed:
    - Set badge to “Printer: Error”.
- Tooltip on the badge:
  - Clarify that browsers cannot verify actual physical printers; this only indicates whether the browser supports printing.
- When the user clicks “Print”:
  - Attempt to print via `window.print()` on the label sheet.
  - If `print()` throws synchronously:
    - Show a toast error and set badge to “Printer: Error”.
  - Do not claim “Ready” at any point.
