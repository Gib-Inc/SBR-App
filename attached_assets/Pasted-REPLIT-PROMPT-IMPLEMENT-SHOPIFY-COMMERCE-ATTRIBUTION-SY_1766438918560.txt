What the Sync must do (high-level behavior)
When user clicks Shopify → Sync:
A) If no completed backfill exists:
   - Run ONE-TIME BACKFILL:
     1) Pull ALL Shopify orders historically (6K+) using GraphQL paging OR Bulk Operation if implemented
     2) Aggregate per customer: first source/date, last source/date, total purchase count, LTV, all sources set
     3) Match those aggregates to GHL contacts by email/phone and update fields/tags
     4) Save sync checkpoints and mark backfill complete
B) If backfill completed:
   - Run INCREMENTAL SYNC:
     1) Pull orders since last sync checkpoint
     2) Recompute aggregates only for impacted customers
     3) Patch GHL contact(s) for those customers

Aggregation rules (per customer key)
Customer identity keys (strict; avoid fuzzy):
- email_key: prefer order.customer.email, else order.email; lowercase trim.
- phone_key: prefer customer.phone, else shipping/billing phone; normalize to E164.

Aggregation:
- purchase_count: total number of qualifying orders for that customer
- first_order_at/source: minimum createdAt among qualifying orders + its source
- last_order_at/source: maximum createdAt among qualifying orders + its source
- sources_set: set of sources encountered (amazon/shopify/unknown) but store as sorted comma list excluding unknown unless all unknown
- lifetime_value_cents: sum of totalPriceSet amounts for qualifying orders

Qualifying orders:
- v1: include paid orders; exclude canceled/voided if flagged by API fields available.
- If cancellation/refund fields are complicated, at minimum exclude fully canceled orders. Log if uncertain.
