C) BACKORDER LOGIC + GHL OPPORTUNITY
==================================================
INVENTORY / ORDER PROCESSING
- For each order line:
  - Let qtyOrdered = line quantity, qtyOnHand = current inventory for that product.
  - If qtyOrdered <= qtyOnHand:
    - qtyFulfilledFromStock = qtyOrdered.
    - backorderQty = 0.
    - Reduce qtyOnHand by qtyOrdered.
  - If qtyOrdered > qtyOnHand:
    - qtyFulfilledFromStock = qtyOnHand.
    - backorderQty = qtyOrdered - qtyOnHand.
    - Set qtyOnHand = 0.
  - Persist backorderQty per line.

- At order level:
  - Compute totalBackorderQty = sum of backorderQty across all lines.

GHL OPPORTUNITY CREATION
- If totalBackorderQty > 0 for the order:
  - Immediately create ONE GHL Opportunity for the order.
  - Use env/config:
    - GHL_API_KEY (or existing credential config).
    - GHL_PROD_PIPELINE_ID.
    - GHL_PROD_STAGE_ALERTED_ID.
  - Use existing GHL client/utilities (do not duplicate) to:
    - Create an opportunity in pipeline GHL_PROD_PIPELINE_ID at stage GHL_PROD_STAGE_ALERTED_ID.
    - Link to the correct GHL contact for the customer.
    - Include in description or custom fields:
      - Order id.
      - For each short item: product name/SKU and backorderQty.
  - Store the created opportunity id in ghlProductionOpportunityId on the Order.
