1. OVERVIEW OF NEW REFUND RULES
====================================

Business rules for refunds:

1) When a customer requests a refund:
   - GHL → calls our Replit refund endpoint (you already created this: POST /api/ghl/custom-actions/create-return-label).
   - Replit → calls Shippo to create a RETURN label and gets the final UPS rate we will pay.
   - Replit → calculates the AVAILABLE REFUND AMOUNT like this:

      - totalReceived = total amount our company actually received for that order (order total AFTER tax; use the field you already store for “total paid by customer”).
      - shippingCost = the UPS rate for the return label (the amount Shippo tells us).
      - labelFee = flat $1 fee we charge for the label.
      - totalReturnCost = shippingCost + labelFee.
      - baseRefundAmount = totalReceived - totalReturnCost.
      - If baseRefundAmount < 0, clamp it to 0.

2) Damage rules when the product comes back to the warehouse:
   - When warehouse scans the returned items, we no longer use a flat 80% vs 100%.
   - Instead, we calculate damage PER DAMAGED STOCK ITEM:
     - For each damaged line item in the return:
       - damageAmountForItem = 10% of that item’s line total (unit price * quantity for that line).
     - damageDeductionTotal = sum of all damageAmountForItem for all damaged items on this return.
   - finalRefundAmount = baseRefundAmount - damageDeductionTotal.
   - If finalRefundAmount < 0, clamp it to 0.

Example:
- totalReceived = $60
- shippingCost (UPS) = $9
- labelFee = $1
- baseRefundAmount = 60 - (9 + 1) = $50
- If one damaged part has line total $60:
  - damageDeductionTotal = 10% of 60 = $6
  - finalRefundAmount = 50 - 6 = $44

We must store and surface both the baseRefundAmount (what we tell the customer they are “eligible for”) AND the finalRefundAmount (after damage is assessed), plus a breakdown of why.
